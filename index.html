1 Fun With Type Coercion

1.1 Examples of Type Coercion

'one' + 1 // 'one1'
true + 1 // 2
[] + [] // ''
[] + {} // [object Object]
{} + {} // NaN

1.2 How Type Coercion Works

1.2.1 ECMAScript Spec Abstract Operations

  • ToNumber
  • ToString
  • ToBoolean
  • etc.

1.2.2 Values Are Converted To Primitives With ToPrimitive()

  • ToPrimitive(input [, PreferredType?])
  • if input is a primitive, return input
  • if input.valueOf() is a primitive, return that
  • if input.toString() is a primitive, return that
  • otherwise, throw a TypeError

1.2.3 Seeing ToPrimitive() in action

var obj = {
       valueOf: () => {
           console.log("valueOf method called");
           return {}; // not a primitive
       },
       toString: () => {
           console.log("toString method called");
           return {}; // not a primitive
       }
   }

Number(obj)
// valueOf method called
// toString method called
// TypeError

String(obj)
// toString method called
// valueOf method called
// TypeError

var obj = {
       valueOf: () => {
           console.log("valueOf method called");
           return 2; // primitive!
       },
       toString: () => {
           console.log("toString method called");
           return {}; // not a primitive
       }
   }

Number(obj)
// valueOf method called
// 2

String(obj)
// toString method called
// valueOf method called
// "2"

var obj = {
       valueOf: () => {
           console.log("valueOf method called");
           return {}; // not a primitive
       },
       toString: () => {
           console.log("toString method called");
           return 'string'; // primitive!
       }
   }

Number(obj)
// valueOf method called
// toString method called
// NaN

String(obj)
// toString method called
// "string"

1.2.4 Stepping though a conversion

'one' + 1 // 'one1'
  • + fails to coerce 'one' into a number
  • + convert 1 into a string
  • + joins 'one' and '1'.
true + 1
  • + converts true into a number, 1
  • + adds 1 and 1

1.3 Using Type Coercion for Good

1.3.1 Default Assignment Shorthand

Without type coercion, values need to be explicitly checked. 

function myFunc(input) {
  var result = someFunc(input);

  if (result !== undefined && result && null) {
    return result;
  } else {
    return 0;
  }
}

With type coercion, non-boolean values can be checked with boolean operators. 

function myFunc(input) {
  return someFunc(input) || 0;
}
